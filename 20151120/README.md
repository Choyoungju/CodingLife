Albocede DNA 문제: https://code.google.com/codejam/contest/10214486/dashboard#s=p3

라스칼라코딩단의 번역
```
에일리언 A종의 DNA는 핵산 a, b, c, d로 구성되어 있습니다. 에일리언 A들은 각자 다른 순서의 헥산들을 가질 수 있습니다. 에일리언 A의 DNA 헥산의 순서열은 다음과 같은 규칙을 따라 구성되어 있습니다.
- 최소 하나 이상의 a, b, c, d를 가집니다.
- 모든 a는 b앞에 오며, 모든 b는 c앞에, 모든 c는 d앞에 옵니다.
- a와 c의 개수는 동일합니다.
- b와 d의 개수는 동일합니다.

예를 들면, abcd나 aabbbccddd는 에일리언 A의 올바른 DNA 헥산 순서입니다. acbd, abc, abbccd는 올바르지 않습니다.
에일리언 A-n은 에일리언 A에서 진화한 종입니다. 에일리언 A-n의 DNA 헥산 열은, 하나 이상의 에일리언 A의 DNA 헥산열의 끝과 끝을 연결하여 만들어집니다.
예를 들어, abcd 나, aaabcccdaabbbccdddabcd는 올바른 에일리언 A-n의 DNA입니다. (올바른 에일리언 A-n의 DNA가, 에일리언 A의 DNA조건을 만족시켜야 하지는 않습니다.)

에일리언 탐험에서, 당신은 a, b, c, d로 구성된 흥미로운 DNA조각을 복구해야 합니다. 당신은 에일리언 A-n의 DNA조건을 만족하는 헥산열이 이 조각에서 얼마나 많은 부분 헥산열로 나타날지에 대해 관심이 있습니다. (만일 여러개의 다른 부분에서 얻어낸 부분 헥산열이 동일할지라도, 하나의 헥산열로 가정하지 않습니다.) 만일 결과가 너무 크다면 1000000007 (=10^9 + 7)로 나눈 나머지를 구해주세요.

부분 헥산열에 대해서는 아래를 참고하세요.
https://en.wikipedia.org/wiki/Subsequence

[입력]
입력의 첫번째 줄은 문제 개수 T를 나타냅니다. 이후 T개의 줄에 a, b, c, d로만 구성된 문자열 S가 주어집니다.

[출력]
각각의 테스트 케이스에서, 출력은 "Case #x: y" 형식의 한줄입니다. x에는 1부터 시작하는 테스트 케이스 번호가 들어가며, y에는 x번째 테스트 케이스의 결과가 들어갑니다.

[제한]
1 <= T <= 20

작은 테스트 데이터 집합
1 <= S의 길이 <= 50

큰 테스트 데이터 집합

1 <= S의 길이 <= 500

[예제]

입력
5
abcd
aaaabcd
aaaabbccd
abcdabcdaabccd
b

출력

Case #1: 1
Case #2: 4
Case #3: 28
Case #4: 71
Case #5: 0
```
